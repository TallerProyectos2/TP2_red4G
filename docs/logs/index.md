# Logs Index

Use the directories under `docs/logs/` for lightweight operational evidence.

## Directories

- `docs/logs/changelog/`
  - short notes describing meaningful system changes

- `docs/logs/validations/`
  - results of runtime checks and end-to-end tests

- `docs/logs/incidents/`
  - failures, unexpected states, or recovery notes

## Logging Rule

Add a log entry when:

- a multi-step technical change was completed,
- a validation produced meaningful evidence,
- a blocker or incident changed the plan.

The logs are supporting evidence. `PLAN.md` and Jira remain the primary planning layers.
